# Stakeholder Onboarding Guide: LRMC Prompt Library

Welcome! This guide helps Legal, Regulatory, and Compliance (LRMC) professionals understand how to use and contribute to the Prompt Engineering Library.

## Purpose

This library supports AI agents that automate:
- Clause triage
- Document review
- Policy summarization
- AML screening
- Compliance tracking

## What Is a Prompt?

A prompt is a structured instruction given to an AI model. It defines:
- The role the model plays
- The task it performs
- The format of its output

## How to Browse Prompts

1. Visit the GitHub repo: `https://github.com/[your-repo-name]`
2. Open the `prompts/` folder
3. Use the `prompt-data-dictionary.csv` to filter by domain or use case

## How to Submit Feedback

- Open a GitHub Issue with:
  - Prompt ID
  - Description of issue
  - Suggested improvement
- Or email the Prompt Library Owner

## How to Request a New Prompt

1. Describe the task or workflow
2. Provide sample inputs and desired outputs
3. Submit via GitHub Issue or internal request form

## Governance & Review

All prompts are reviewed for:
- Accuracy
- Bias and hallucination risk
- Regulatory alignment
- Stakeholder usability
